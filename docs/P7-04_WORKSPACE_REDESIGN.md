# P7-04 · TOHOSHOU AI Workspace 重构方案

> 纯产品结构设计，不改代码 / 不部署 / 不删除 / 不讨论实现。日期 2026-07-16。
> 目标：把产品从「开发者视角（按页面堆砌）」改为「角色工作流（按人使用）」。基于 P7-00~03 真实现状。

---

## 一、重新定义角色（按人，不按页面）

| 角色 | 是谁 | 每天要回答的问题 | 绝不关心 |
|---|---|---|---|
| **① 老板 Boss** | 每天做投资决策的人 | 今天做不做？买哪只？持仓怎么办？为什么？ | 模型内部/因子/Cron/数据健康/版本 |
| **② 管理员 Admin** | 保系统活着的人 | 数据同步了吗？Cron 跑了吗？部署成功吗？健康吗？ | 选哪只股/买卖时机 |
| **③ 研究 Research** | 调模型/验证 Alpha 的人 | 因子有效吗？V3 能上吗？策略跑赢吗？学到了什么？ | 日常下单/持仓 |

**关键洞察**：本系统实际由**同一个人**兼任三角色，但**三种心智模式完全不同**。老板模式要「快、少、直达结论」；研究模式要「全、深、可对比」。把三者混在一个导航里，就是当前"37 页混乱"的根因——**不是页面太多，是没有按心智分区**。

## 二、老板工作区（95% 日常在 4 个入口完成）

### 老板登录后应该看到（且仅看到）：
```
① 决策中心   → 今天做不做 + 早盘五选 + 关注池 + 收盘决策（含首页"今日建议"落地）
② 股票中心   → 买哪只（选股/行业/主题/产业链/新闻/指标）+ 个股详情
③ 策略       → 三策略今日推荐（Day/Swing/Long，只看"推荐什么"，不看回测细节）
④ 我的持仓   → Paper 组合/持仓/我的自选/操作建议
```

### 老板为什么需要打开这些？——答案全是「不需要」
| 页面 | 老板需要吗 | 处置 |
|---|---|---|
| 学习报告 | ❌ 模型学习内部，与"今天买什么"无关 | 移出→研究区 |
| 版本中心 | ❌ 模型版本/部署审计 | 移出→研究/管理区 |
| 实验 | ❌ 实验登记（且表为空） | 移出→研究区(隐藏) |
| Runtime | ❌ 流水线/GPT 日志 | 移出→管理区 |
| Verify | ❌ 数据完整性校验 | 移出→管理区 |
| Sync | ❌ 数据同步操作 | 移出→管理区 |
| 因子中心/晋升/平台 | ❌ 因子工程 | 移出→研究区 |
| 回测(明细) | ❌ 老板只需知道"策略有效"，不需看逐日回测 | 移出→研究区(策略页留一句"胜率/近月表现"摘要即可) |

→ **老板工作区 = 4 个入口，零技术页面，零内部指标。** 技术内容一律"查看详情"按需跳研究/管理区。

## 三、管理员工作区（老板默认不可见）

**职责**：数据 / Cron / 部署 / 日志 / 校验 / 同步 / Runtime / 健康。统一为一个「系统管理」工作区：
```
① 系统总览(Mission Control) → Health/Cron/Web/DB/部署/版本一屏
② 数据同步(Sync)            → J-Quants/新闻/TDnet 同步状态与触发
③ 数据校验(Verify)          → 完整性/回测覆盖/评分校验
④ Runtime/日志              → 流水线阶段 + GPT Runtime 日志
⑤ 部署历史(Deployments)     → 部署记录/回滚参考
⑥ Cron/健康                 → 定时任务时刻表 + Data Health Guard
```
6 个入口，纯运维，老板永不进入。

---

## 四、研究工作区（所有模型/Alpha/Shadow 统一，不分散）

**职责**：因子/Alpha/Fusion/Regime/V3/Shadow/回测/学习/版本/实验。统一为一个「研究中心」工作区：
```
① 研究综合         → 因子健康/Alpha/regime 一屏概览
② 因子             → 注册库 + 晋升 + 平台/Integrity（现 3 页合一）
③ Alpha            → 影子评分 + 因子分析 + Alpha 回测
④ 市场状态 Regime  → regime 时间线
⑤ Fusion           → 融合策略研究 + Fusion 纸面
⑥ Scoring V3 ⚫    → shadow / calibration / freeze（未上线引擎观察）
⑦ 学习报告         → AI 学习总结
⑧ 版本 / 实验      → 版本快照 + 实验登记
⑨ 策略回测(深)     → StrategyBacktest + AlphaBacktest 双源
```
8-9 个入口，深度可对比，老板默认不可见。

## 十二、是否引入 Workspace（工作区）模式 —— 明确推荐

### ✅ 推荐引入「轻量 Workspace 模式」，并作为 P7 最终架构。

**为什么值得：**
1. **根治混乱**：当前 20→7 导航（B-1）只是"折叠 admin"，治标；Workspace 是"按心智彻底分区"，治本。老板永不撞见因子/Cron/V3。
2. **可扩展**：未来任何新研究工具进"研究区"、新运维进"管理区"，**永远不污染老板区**——解决"新功能持续堆积"的长期病根（P7 立项初衷）。
3. **已有地基**：B-1 的统一 nav-config 已带 `adminOnly` 分区标记，升级为 `workspace` 三态是自然演进，非推倒重来。
4. **零权限负担**：本系统单人运营，用**软切换器**（非登录鉴权）即可——默认老板、一键切管理员/研究。不需要账号/角色系统，落地轻。

**为什么不做重的（不引入硬鉴权 Workspace）：**
- 同一个人兼三角色，硬权限门反而添堵；软切换器已足够达成"默认清爽、按需展开"。

### Workspace 作为 P7 最终架构的收束意义
- **B-2/B-3 已完成的决策/研究 hub** = 老板工作区的两大支柱。
- **未做的 B-4/B-5/B-6** 在 Workspace 框架下**重新定义**为：
  - B-4 回测 → **不进老板区**，进研究工作区（老板策略页只留摘要）
  - B-5 因子三页/学习/版本/实验 → **全进研究工作区**；我的自选进老板持仓
  - B-6 运维三页 → **全进管理员工作区**
- 即：**Workspace 模式让 B-4/5/6 从"往老板 nav 塞 Tab"变成"归位到管理员/研究工作区"**，老板区反而更干净。

### 一句话结论
> **引入轻量三工作区（老板/管理员/研究，软切换、默认老板）。它是 P7 的正确终点：老板每天只面对 4 个入口，管理员与研究各自独立成区，产品从此按"人怎么用"生长，而非按"开发者加了什么"堆积。**
